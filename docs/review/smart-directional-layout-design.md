# 智能方位布局设计

## 需求

1. **人类玩家固定在南边**（视觉上始终在底部）
2. **根据打牌顺序（顺时针）智能调整方位**
   - 下家（顺时针下一个）→ 西
   - 对家/队友（对面）→ 北  
   - 上家（顺时针上一个）→ 东

## 打牌顺序

顺时针顺序：0 → 1 → 2 → 3 → 0

## 方位分配逻辑

### 人类玩家在索引0

- 人类玩家（索引0）→ 南（固定，不显示）
- 下家（索引1，顺时针下一个）→ 西
- 对家（索引2，对面）→ 北
- 上家（索引3，顺时针上一个）→ 东

### 人类玩家在索引1

- 人类玩家（索引1）→ 南（固定，不显示）
- 下家（索引2，顺时针下一个）→ 西
- 对家（索引3，对面）→ 北
- 上家（索引0，顺时针上一个）→ 东

### 人类玩家在索引2

- 人类玩家（索引2）→ 南（固定，不显示）
- 下家（索引3，顺时针下一个）→ 西
- 对家（索引0，对面）→ 北
- 上家（索引1，顺时针上一个）→ 东

### 人类玩家在索引3

- 人类玩家（索引3）→ 南（固定，不显示）
- 下家（索引0，顺时针下一个）→ 西
- 对家（索引1，对面）→ 北
- 上家（索引2，顺时针上一个）→ 东

## 通用算法

对于人类玩家在索引 `humanIndex`（玩家总数4）：

```
人类玩家 → 南（固定，不显示）

下家 = (humanIndex + 1) % 4 → 西（右侧）
对家 = (humanIndex + 2) % 4 → 北（顶部）
上家 = (humanIndex + 3) % 4 → 东（左侧）
```
