# Chrome开发者工具使用指南

## 如何打开Chrome开发者工具

### 方法1：快捷键（最快）
- **Windows/Linux**: 按 `F12` 或 `Ctrl + Shift + I`
- **Mac**: 按 `Cmd + Option + I`

### 方法2：右键菜单
1. 在页面上右键点击
2. 选择"检查"或"Inspect"

### 方法3：菜单栏
1. 点击Chrome右上角的三个点（菜单）
2. 选择"更多工具" → "开发者工具"

## 查看控制台（Console）

控制台会显示：
- **错误信息**（红色）
- **警告信息**（黄色）
- **日志信息**（console.log）
- **性能问题**

### 查看错误
1. 打开开发者工具（F12）
2. 点击"Console"（控制台）标签
3. 查看红色错误信息

### 常见错误类型
- **JavaScript错误**: 代码执行错误
- **网络错误**: API请求失败
- **资源加载错误**: 文件找不到

## 查看性能（Performance）

### 查看CPU使用情况
1. 打开开发者工具
2. 点击"Performance"（性能）标签
3. 点击录制按钮（圆点）
4. 操作游戏
5. 停止录制
6. 查看CPU使用情况

### 查看内存使用
1. 打开开发者工具
2. 点击"Memory"（内存）标签
3. 可以查看内存使用情况

## 查看网络请求（Network）

1. 打开开发者工具
2. 点击"Network"（网络）标签
3. 查看所有网络请求
4. 检查是否有请求失败或很慢

## 调试技巧

### 1. 查看MCTS计算时间
在控制台输入：
```javascript
// 查看当前游戏状态
console.log('当前游戏状态:', window.gameState);
```

### 2. 检查是否有无限循环
- 查看CPU使用率是否持续100%
- 查看控制台是否有重复的错误信息

### 3. 检查内存泄漏
- 打开Memory标签
- 多次操作游戏
- 查看内存是否持续增长

### 4. 暂停执行
- 在代码中设置 `debugger;` 语句
- 或者点击Sources标签，设置断点

## 常见问题排查

### 问题：游戏很慢，AI不出牌
**可能原因**：
1. MCTS迭代次数太高
2. 模拟深度太深
3. 有无限循环

**解决方法**：
1. 查看控制台错误
2. 检查CPU使用率
3. 降低MCTS迭代次数（已优化）

### 问题：语音不工作
**检查**：
1. 控制台是否有错误
2. 浏览器是否支持语音合成
3. 是否允许了音频权限

### 问题：页面卡死
**检查**：
1. 查看Performance标签
2. 检查是否有长时间运行的脚本
3. 查看内存使用情况

## 快速调试命令

在控制台（Console）中可以输入：

```javascript
// 查看当前游戏状态
console.log('游戏状态:', window.gameState);

// 停止所有语音
window.speechSynthesis.cancel();

// 检查MCTS配置
console.log('MCTS配置:', {
  iterations: 300,
  simulationDepth: 50
});
```

## 性能优化建议

如果游戏还是很慢，可以：

1. **降低MCTS迭代次数**：在代码中修改 `mctsIterations` 为更小的值（如100-200）
2. **使用简单AI策略**：在游戏设置中选择"智能策略算法"而不是"MCTS"
3. **减少玩家数量**：4人游戏比8人游戏快很多
4. **关闭语音**：如果语音导致问题，可以暂时关闭
